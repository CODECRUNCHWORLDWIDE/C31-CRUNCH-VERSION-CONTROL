# C31 · Crunch Version Control — Syllabus

**Format:** 8 weeks · self-paced · open-source (GPL-3.0). Each week = 3 lectures, 3 exercises, 2 challenges, a mini-project, homework, and a quiz.

**Prerequisites:** You can run commands in a terminal and have written some code in any language. No prior Git knowledge assumed.

**Assessment (honor-based):** tick each lesson complete in the reader; finish the weekly mini-project and quiz. A certificate is issued on 100% completion to One-Stop members.

| Week | Theme | Mini-project |
|------|-------|--------------|
| 1 | Git fundamentals — objects, the three trees, staging | Reconstruct a repo's history by hand |
| 2 | Branching & merging — conflicts, fast-forward vs. three-way | Merge three feature branches cleanly |
| 3 | Remotes & GitHub — push/pull/fetch, forks | Fork, contribute, and sync |
| 4 | Rewriting history — rebase, amend, cherry-pick, reflog | Clean up a messy branch for review |
| 5 | Pull requests & review — GitHub flow | Ship a feature via a reviewed PR |
| 6 | CI with GitHub Actions — tests + status checks | A repo where broken code can't merge |
| 7 | Advanced Git — hooks, bisect, worktrees, releases | Find a bug with `git bisect` + cut a release |
| 8 | Team workflows at scale — trunk-based, protected branches | Design + run a small team's workflow (capstone) |

**Outcome:** run a protected, CI-gated repository a real team could ship from.

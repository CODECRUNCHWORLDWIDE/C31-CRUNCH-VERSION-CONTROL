# Week 8 — Exercises

Three guided reps, ~45–55 min each. Do them in order — each configures a piece of the workflow you'll assemble in the capstone. Type the commands yourself; don't paste blindly.

1. **[Exercise 1 — Protect and own](exercise-01-protect-and-own.md)** — turn on branch protection and add a `CODEOWNERS` file, then prove an unreviewed change can't merge.
2. **[Exercise 2 — Choose a strategy](exercise-02-choose-a-strategy.md)** — read three team scenarios and pick + justify a branching strategy for each.
3. **[Exercise 3 — Automate a release](exercise-03-automate-a-release.md)** — wire a workflow that computes a version, generates a changelog, and cuts a tagged release from Conventional Commits.

## Suggested workflow

- Use a **throwaway repo** you own for the config exercises — you'll be locking `main` down and reverting.
- A second GitHub account or an org with a teammate makes the review parts realistic. Solo? Each exercise has a solo path in its notes.
- Keep the [lecture notes](../lecture-notes/) open beside you; every exercise maps to one lecture.

The point isn't to finish fast — it's to have *run* each control at least once, so in the capstone you're assembling things you've already touched.

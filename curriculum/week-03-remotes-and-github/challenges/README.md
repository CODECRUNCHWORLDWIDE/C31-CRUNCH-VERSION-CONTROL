# Week 3 — Challenges

Two open-ended problems. Unlike the exercises, these have no single right sequence of commands — the point is to diagnose, decide, and justify your approach.

1. **[Recover from a rejected push](challenge-01-recover-rejected-push.md)** — your push bounces with "non-fast-forward." Fix it without destroying anyone's work. The wrong fix (`--force`) is the whole lesson.
2. **[Contribute through a fork](challenge-02-contribute-through-a-fork.md)** — take a real change all the way from fork to an open pull request against an upstream repo.

Optional but strongly recommended — the gap between "can run Git commands" and "trusted with a shared repository" lives right here.

## How these are judged

There's no autograder. You'll write a short `writeup.md` for each explaining what happened, what you tried, and why your final approach was safe. A good writeup shows you *understood the state Git was in* — not just that you found commands that made the error go away.

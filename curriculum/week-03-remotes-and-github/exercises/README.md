# Week 3 — Exercises

Three exercises, ~40 minutes each. Type the commands, don't paste — the goal is muscle memory, not a finished checklist.

1. **[Exercise 1 — Clone and push](exercise-01-clone-and-push.md)** — clone a repo and push your first commit to a remote. Uses a local "server" so nothing depends on the network.
2. **[Exercise 2 — Track and pull](exercise-02-track-and-pull.md)** — set up upstream tracking, then practice `fetch` vs. `pull` deliberately.
3. **[Exercise 3 — Fork and sync](exercise-03-fork-and-sync.md)** — fork a real GitHub repo, wire up `upstream`, and sync it. (Needs a GitHub account.)

Run them in order — each builds on the last. Exercises 1 and 2 work fully offline; Exercise 3 uses your GitHub account.

## Suggested workflow

- Open a fresh terminal.
- Open the exercise file alongside in your editor.
- Type each command yourself and **read the output before moving on**.
- When a command surprises you, stop and run `git status` and `git branch -vv` to see the state before continuing.
- Keep a `notes.md` open; jot the one-line answer to each "explain in a sentence" prompt.

## What you'll be able to do after all three

- Clone, commit, and push without looking anything up.
- Explain the difference between `fetch` and `pull` by having *felt* it.
- Fork, add `upstream`, and sync a fork — the foundation for this week's challenges and mini-project.
